# 1-Week C++ Interview Crash Plan

Goal: maximize interview readiness in 7 days using high-yield revision and short practice loops.

## Daily Structure (repeat every day)

- 90 min: concept revision from fast-track notes
- 60 min: coding practice (2 easy + 1 medium)
- 30 min: explain one topic aloud (mock interview mode)
- 30 min: fix weak points and make a small cheat sheet

## Day-wise Plan

### Day 1 - Core Syntax + Arrays + Strings + Functions

- Use Module1 fast-track: arrays, strings, pointers basics, pass by value/ref
- Practice: array traversal, frequency map, two-sum, reverse string
- Interview drill: difference between array and vector; value vs reference

### Day 2 - Pointers + Memory + Recursion

- Revise pointer arithmetic, `new/delete`, stack vs heap, recursion tree
- Practice: reverse linked style array via pointers, recursion basics, factorial/fibonacci with memoization
- Interview drill: memory leak, dangling pointer, base condition design

### Day 3 - OOP + Pillars + Polymorphism

- Revise class design, constructor/destructor, virtual functions, abstraction
- Practice: small class design with interface + derived class
- Interview drill: runtime polymorphism, virtual destructor, override vs overload

### Day 4 - Inheritance + Encapsulation + SOLID mindset

- Revise inheritance types, access control, object slicing, composition vs inheritance
- Practice: implement base/derived with pure virtual API
- Interview drill: diamond problem, when to prefer composition

### Day 5 - STL + Complexity + Coding Patterns

- Revise vector/map/unordered_map/set/stack/queue + complexity
- Practice: sliding window, hash map, sorting + binary search
- Interview drill: ordered vs unordered containers and tradeoffs

### Day 6 - Modern C++ (RAII, smart pointers, move semantics, PIMPL)

- Use Module6 fast-track heavily
- Practice: implement one RAII wrapper + one PIMPL sample
- Interview drill: Rule of 5, unique_ptr ownership transfer, ABI stability

### Day 7 - Mock Interview Day

- 60 min DSA mixed set (timed)
- 60 min OOP/system design discussion (speak your thought process)
- 60 min C++ rapid-fire theory
- Final 30 min: revise mistake log only

## Must-Know Rapid-Fire (high probability)

- `struct` vs `class`
- stack vs heap
- shallow vs deep copy
- Rule of 3/5/0
- `unique_ptr` vs `shared_ptr` vs `weak_ptr`
- `virtual` and virtual destructor
- `override` and `final`
- lvalue vs rvalue and `std::move`
- `std::vector` growth and iterator invalidation
- `map` vs `unordered_map`
- RAII and exception safety
- PIMPL: why and when

## Last 24 Hours Strategy

- Do not learn new topics.
- Revise cheat sheets + mistake log.
- Solve 3 medium questions max with full explanations.
- Sleep well; interview clarity beats cramming.
